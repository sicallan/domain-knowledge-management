PROJECTION LOGIC:
- The flow's steps are held as **ordered step ids in the OrchestrationFlow's `steps` array** (NOT a membership edge — the L3 schema stores `steps: string[]`). Resolve each via `getEntry`, then order by the step's `sequence` field.
- `emits`: the events a step emits — read the `emits` edges (OrchestrationStep → Event) out of each step. `transitions`: the `transitionsTo` edges (OrchestrationStep → StateTransition); surface fromState/toState/guardCondition from the StateTransition node. `consumes`: best-effort — derive from the flow's trigger Event (the `triggers` Event → OrchestrationFlow edge) on the first step, else empty; keep it additive, do not over-fit.
- **Decision points (criterion 3):** a step with an `invokes` edge (OrchestrationStep → Decision) is `isDecisionPoint: true` and carries `decision { id, name, type, outcomes }`. Read the Decision node's axis field **`decisionType`** (automated|manual|hybrid) and map it to the view's `type` field — NOTE the Decision schema uses `decisionType`, NOT `type` (`type` is the base-entry discriminator const; same convention as Event's `eventType`). The Decision's `outcomes: string[]` → the outcome branches (`{ label, producesEventId? }`); set `producesEventId` from a `produces` edge (Decision → Event) when one matches an outcome, else omit.
- **Unknown flow (criterion 4):** `getView('behaviour-flow', { flowId: <missing> })` returns an empty / clearly "not found" result (no exception) — follow the Phase 1 view-engine error contract (see how the engine/DomainMapProjector handle a missing/empty projection).
- **invalidatedBy (criterion 6):** return true only for a `GraphMutationEvent` that touches this flow's steps / events / state-transitions / decisions; mirror `DomainMapProjector.invalidatedBy`. Declare the refresh policy (on-demand is fine — the flow view is cheap/scoped; spec 007 Decision 1).
